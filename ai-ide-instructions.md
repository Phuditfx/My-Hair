## 🛠️ 2. ขั้นตอนและแนวทางในการแก้ไขหรือเพิ่มฟีเจอร์ (Modification & Feature Addition Protocol)
เมื่อผู้ใช้สั่งให้คุณแก้ไขหรือเพิ่มฟีเจอร์ใดๆ ให้ปฏิบัติตามลำดับขั้นตอนดังนี้:

1. **วิเคราะห์โค้ดเดิม (Analyze Existing Code):** อ่านโค้ดในไฟล์ที่เกี่ยวข้องก่อนเสมอเพื่อทำความเข้าใจสถาปัตยกรรม และตรวจสอบว่าการแก้ไขจะไม่กระทบฟังก์ชันอื่น (Breaking Changes)
2. **รักษาโครงสร้างความปลอดภัย (Maintain RBAC):** ตรวจสอบเสมอว่าฟีเจอร์นั้นๆ สอดคล้องกับสิทธิ์การเข้าถึงหรือไม่:
   - **Pending:** ห้ามเห็นหน้าหลักเด็ดขาด
   - **VIP:** เข้าถึงสูตรสีและบทเรียน VIP ได้ แต่ห้ามใช้ AI Gen
   - **Admin (phudit.mahawongsanan@gmail.com / phudit.fx@gmail.com):** เข้าถึงได้ทุกฟีเจอร์รวมถึงระบบ AI
3. **รองรับ Responsive Design เสมอ:** หน้าตา UI ที่แก้ไขหรือสร้างใหม่ต้องแสดงผลได้อย่างสมบูรณ์แบบบน **PC, Tablet (iPad), และ Mobile** (เนื่องจากช่างผมต้องเปิดดูหน้ากระจกผ่านแท็บเล็ตและมือถือ)
4. **คงธีม Dual Workspace:** โค้ดที่สร้างใหม่ต้องรองรับเงื่อนไขการสลับโหมด:
   - **Salon Mode:** ใช้ธีมสีอ่อน (Light Theme)
   - **Barber Mode:** ใช้ธีมสีเข้ม (Dark Theme)

## 🧪 3. ขั้นตอนการตรวจสอบและทดสอบระบบ (Strict Verification & Testing Protocol)
**[สำคัญที่สุด]** หลังจากเขียนโค้ดหรือแก้ไขเสร็จแล้ว ห้ามส่งงานทันที คุณต้องรันกระบวนการตรวจสอบและทดสอบดังต่อไปนี้ก่อนเสมอ:

1. **Linting & Compile Check:** - รันคำสั่งตรวจสอบเพื่อหา Error (เช่น `npm run lint` หรือตรวจสอบประเภทตัวแปรใน TypeScript)
   - ตรวจสอบว่าไม่มี Syntax Error หรือ Missing Import หลงเหลืออยู่
2. **UI & Responsive Check:**
   - ตรวจสอบการจัดวาง Component ในหน้าจอขนาดต่างๆ (Mobile Width: ~375px, Tablet Width: ~768px, Desktop Width: >1024px)
   - มั่นใจว่าไม่มี Element ใดเบลอ หลุดขอบ หรือทับซ้อนกัน
3. **Functional Check:**
   - ทดสอบจำลองการทำงาน (Mock Test) ตามสิทธิ์ของ User แต่ละระดับว่าทำงานถูกต้องตรงตามเงื่อนไขหรือไม่

## 🚀 4. ขั้นตอนการบันทึกและอัปเดตขึ้น GitHub (GitHub Version Control Protocol)
เมื่อโค้ดผ่านการตรวจสอบตามข้อ 3 แล้ว ให้คุณดำเนินการจัดเก็บซอร์สโค้ดและส่งขึ้น GitHub โดยอัตโนมัติผ่าน Terminal/Git Tool ของคุณตามขั้นตอนดังนี้:

1. **ตรวจสอบสถานะไฟล์ (Git Status):**
   - เช็กไฟล์ที่มีการเปลี่ยนแปลงทั้งหมด
2. **เพิ่มไฟล์เข้าระบบเตรียม Commit (Git Add):**
   - รันคำสั่ง `git add .` เพื่อเตรียมบันทึกไฟล์ทั้งหมด
3. **บันทึกประวัติการแก้ไข (Git Commit):**
   - เขียน Commit Message ให้ชัดเจน กระชับ และสื่อความหมายตามสไตล์มาตรฐานสากล เช่น:
     - `feat: add AI color analysis module for Admin`
     - `fix: resolve responsiveness issue on mobile navbar in Barber mode`
     - `style: adjust light theme background color for Salon workspace`
4. **ส่งโค้ดขึ้น GitHub (Git Push):**
   - รันคำสั่ง `git push origin [ชื่อสาขา/branch ปัจจุบัน เช่น main หรือ master]`
5. **รายงานผลลัพธ์:**
   - สรุปให้ผู้ใช้ทราบว่ามีไฟล์ใดได้รับการแก้ไขบ้าง พร้อมแนบข้อความ Commit Message และแจ้งสถานะการ Push โดนสมบูรณ์
